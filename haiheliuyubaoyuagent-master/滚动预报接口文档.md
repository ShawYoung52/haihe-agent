# 滚动预报接口调用说明

## 一、接口概述

**接口名称**：滚动预报查询（rolling_forecast）

**接口用途**：根据经纬度坐标查询指定站点的未来天气预报数据，支持多种气象要素的多时段查询。

**接口地址**：
```
http://10.226.120.112:8088/tjgrid/gdyb/getGdybDataByParam
```

**请求方法**：HTTP GET

**数据格式**：JSON

---

## 三、参数详解

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|---------|------|------|--------|------|
| **fcstTime** | string | 是 | 动态生成 | 预报起报时间，格式 `YYYYMMDDHHMMSS`，详见"四、起报时间规则" |
| **element** | string | 是 | - | 查询的气象要素代码，多个用英文逗号分隔，详见"六、气象要素映射" |
| **lon** | string | 是 | - | 经度列表，多个用英文逗号分隔，顺序需与 `lat` 一一对应 |
| **lat** | string | 是 | - | 纬度列表，多个用英文逗号分隔，顺序需与 `lon` 一一对应 |
| **mode** | string | 否 | `GDMODE` | 预报模式，默认值为 `GDMODE` |
| **startPeriod** | string | 否 | `0` | 起始预报时段，从起报时间开始算起的第几个时段，默认 `0` |
| **endPeriod** | string | 否 | `240` | 结束预报时段，与 `interval` 组合决定预报总时长，默认 `240` |
| **interval** | string | 否 | `12` | 时间步长（小时），默认 `12`，即每 12 小时一个预报点 |
| **count** | string | 否 | `0` | 计数，默认值 `0` |
| **stationType** | string | 否 | `3` | 站点类型，默认值 `3` |

> ⚠️ **重要说明**：`lon` 和 `lat` 的数量必须完全一致，且每个位置的经纬度要精确配对。例如查询 3 个站点就传 3 个经度和 3 个纬度。

---

## 四、起报时间（fcstTime）规则

系统每天仅在两个时间点生成一次完整预报数据，因此 `fcstTime` 参数**必须**是以下两个时刻之一，否则接口可能返回空值或 `9999.0` 等无效数据。

### 4.1 时间点说明

| 时间点 | 含义 | 生成频率 |
|---------|------|---------|
| `YYYYMMDD080000` | 当天 08:00 起报 | 每天一次 |
| `YYYYMMDD200000` | 当天 20:00 起报（前一天的晚间预报） | 每天一次 |

### 4.2 自动选择逻辑

在代码中，根据当前系统时间自动选择合适的起报时间：

```python
from datetime import datetime, timedelta

now = datetime.now()

if now.hour >= 8:
    # 已过当日 08:00，使用当天 08:00 起报
    fcst_time = now.strftime("%Y%m%d080000")
else:
    # 还未到当日 08:00，使用前一天 20:00 起报
    yesterday = now - timedelta(days=1)
    fcst_time = yesterday.strftime("%Y%m%d200000")
```

### 4.3 举例

| 当前时间 | 自动选择的 fcstTime |
|---------|-------------------|
| 2026-06-11 07:30 | `20260610200000`（前一天 20:00） |
| 2026-06-11 08:00 | `20260611080000`（当天 08:00） |
| 2026-06-11 15:30 | `20260611080000`（当天 08:00） |
| 2026-06-11 21:00 | `20260611080000`（当天 08:00） |
| 2026-06-12 02:30 | `20260611200000`（前一天 20:00） |

> 💡 **建议**：实际使用时，建议先查询近 2-3 个起报时间点，确认哪个有数据。

| 区域名称 | 经度 | 纬度 | 拼接格式 |
|---------|------|------|---------|
| 天津市区 | 117.14 | 39.24 | `117.14_39.24` |
| 蓟州 | 117.45 | 40.05 | `117.45_40.05` |
| 宝坻 | 117.28 | 39.73 | `117.28_39.73` |
| 武清 | 117.06 | 39.43 | `117.06_39.43` |
| 宁河 | 117.85 | 39.38 | `117.85_39.38` |
| 静海 | 116.92 | 38.93 | `116.92_38.93` |
| 北辰 | 117.21 | 39.07 | `117.21_39.07` |
| 西青 | 117.05 | 39.08 | `117.05_39.08` |
| 津南 | 117.42 | 38.95 | `117.42_38.95` |
| 东丽 | 117.34 | 39.08 | `117.34_39.08` |
| 滨海新区 | 117.79 | 39.16 | `117.79_39.16` |

### 5.2 代码中的实现

```python
coord_map = {
    "天津市区": "117.14_39.24",
    "蓟州": "117.45_40.05",
    "宝坻": "117.28_39.73",
    "武清": "117.06_39.43",
    "宁河": "117.85_39.38",
    "静海": "116.92_38.93",
    "北辰": "117.21_39.07",
    "西青": "117.05_39.08",
    "津南": "117.42_38.95",
    "东丽": "117.34_39.08",
    "滨海新区": "117.79_39.16"
}

# 反向映射（用于解析返回结果）
location_map = {v: k for k, v in coord_map.items()}
# 结果: {"117.14_39.24": "天津市区", "117.45_40.05": "蓟州", ...}

def _parse_regions(region_text: str, coord_map: dict):
    """
    根据用户输入的区域文本，返回对应的经度列表和纬度列表
    """
    if "全市" in region_text or region_text == "":
        # 查询全市，使用所有坐标
        coords = list(coord_map.values())
    else:
        # 匹配用户提到的区域
        coords = []
        for region_name in coord_map.keys():
            if region_name in region_text:
                coords.append(coord_map[region_name])
        # 如果都没匹配到，默认查询全市
        if not coords:
            coords = list(coord_map.values())

    # 将 "经度_纬度" 拆分为两个列表
    lons = [c.split('_')[0] for c in coords]
    lats = [c.split('_')[1] for c in coords]
    return lons, lats

# 示例：用户问"北辰明天天气"
lons, lats = _parse_regions("北辰明天天气", coord_map)
# lons = ["117.21"], lats = ["39.07"]

# 示例：用户问"西青和津南后天怎么样"
lons, lats = _parse_regions("西青和津南", coord_map)
# lons = ["117.05", "117.42"], lats = ["39.08", "38.95"]

# 示例：用户问"全市明天天气如何"
lons, lats = _parse_regions("全市", coord_map)
# lons = ["117.14", "117.45", ..., "117.79"] 共11个
# lats = ["39.24", "40.05", ..., "39.16"] 共11个
```

## 六、气象要素（element）映射

`element` 参数用于指定需要查询哪些气象要素。

### 6.1 要素代码表

| 代码 | 中文名称 | 说明 |
|------|---------|------|
| **WEA** | 天气现象 | 如：晴、多云、阴、小雨、雷阵雨等文本描述 |
| **TMAX** | 最高气温 | 时段内的最高气温，单位：℃ |
| **TMIN** | 最低气温 | 时段内的最低气温，单位：℃ |
| **EDA** | 风况 | 包含风向、风速等信息 |
| **RHMAX** | 最大相对湿度 | 最大相对湿度，单位：% |
| **RHMIN** | 最小相对湿度 | 最小相对湿度，单位：% |
| **TCCMAX** | 最大总云量 | 最大总云量 |
| **TCCMIN** | 最小总云量 | 最小总云量 |
| **VISMIN** | 最小能见度 | 最小能见度，单位：米 |
| **TP1H** | 1 小时降水量 | 时段内的累计降水量，单位：mm |

### 6.2 代码中的实现

```python
element_map = {
    "WEA": "天气现象",
    "TMAX": "最高气温",
    "TMIN": "最低气温",
    "EDA": "风况",
    "RHMAX": "最大相对湿度",
    "RHMIN": "最小相对湿度",
    "TCCMAX": "最大总云量",
    "TCCMIN": "最小总云量",
    "VISMIN": "最小能见度",
    "TP1H": "1小时降水量"
}

def _select_forecast_elements(user_message: str):
    """
    根据用户查询智能选择需要查询的气象要素
    """
    # 默认返回基础天气要素（最常用的 7 项）
    selected = ['WEA', 'TMAX', 'TMIN', 'EDA', 'RHMAX', 'RHMIN', 'TP1H']

    # 如果用户特别提到能见度/雾霾
    if any(kw in user_message for kw in ['能见度', '雾', '霾']):
        selected.append('VISMIN')

    # 如果用户特别提到云量
    if any(kw in user_message for kw in ['云量']):
        selected.extend(['TCCMAX', 'TCCMIN'])

    return list(set(selected))  # 去重

# 示例：用户问"明天天气如何"
elements = _select_forecast_elements("明天天气如何")
# 返回: ['WEA', 'TMAX', 'TMIN', 'EDA', 'RHMAX', 'RHMIN', 'TP1H']

# 示例：用户问"明天能见度怎么样，会不会有雾"
elements = _select_forecast_elements("明天能见度怎么样，会不会有雾")
# 返回: ['WEA', 'TMAX', 'TMIN', 'EDA', 'RHMAX', 'RHMIN', 'TP1H', 'VISMIN']
```

> 💡 **经验总结**：
> - 日常天气查询默认使用 7 项基础要素已足够
> - 温度相关只需 TMAX 和 TMIN
> - 降水相关只需 WEA 和 TP1H
> - 雾霾天气建议额外查询 VISMIN

## 七、返回数据结构说明

### 7.1 整体结构

```json
{
  "code": "200",
  "message": "success",
  "resultData": {
    "117.14_39.24": {           // 天津市区
      "WEA": [["晴", "多云", ...]],
      "TMAX": [[28.5, 27.3, ...]],
      "TMIN": [[18.2, 17.5, ...]],
      "EDA": [["东南风3级", "南风4级", ...]],
      "TP1H": [[0.0, 0.0, ...]]
    },
    "117.45_40.05": {           // 蓟州
      "WEA": [["晴", "晴", ...]],
      "TMAX": [[25.0, 24.5, ...]],
      ...
    }
    // ... 其他区域
  }
}
```

### 7.2 关键字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | string | 返回状态码，`200` 表示成功 |
| `message` | string | 返回消息描述 |
| `resultData` | object | 核心数据对象，键为经纬度拼接格式（如 `117.14_39.24`） |
| `resultData[坐标].WEA` | array | 天气现象数组，每个元素是一个时间步的天气描述 |
| `resultData[坐标].TMAX` | array | 最高气温数组，每个元素是一个时间步的气温值（℃） |
| `resultData[坐标].TMIN` | array | 最低气温数组 |
| `resultData[坐标].EDA` | array | 风况数组 |
| `...` | ... | 其他请求的要素 |

### 7.3 无效值处理

接口在某些情况下可能返回以下**无效值**，在使用前需过滤：

| 无效值 | 含义 |
|--------|------|
| `9999.0` | 数值类型的缺失值（气温、降水等） |
| `"--"` | 文本类型的缺失值（天气现象等） |
| `null` | 空值 |

```python
# 数据过滤示例
def is_valid_value(value):
    return value not in [9999.0, "--", None]

# 检查某个数据点
temperature = 25.3
if is_valid_value(temperature):
    print(f"气温: {temperature}℃")  # 正常使用
else:
    print("气温: 暂无数据")          # 或跳过
```

### 7.4 时间序列说明

- **interval=12, startPeriod=0, endPeriod=240** → 每 12 小时一个预报点，共 20 个预报点（240÷12=20）
- 第 0 个时段：从起报时间开始的第 0-12 小时
- 第 1 个时段：从起报时间开始的第 12-24 小时
- 依此类推...

例如起报时间是 06-11 08:00：
- 时段 0：6月11日 08:00 ~ 20:00（白天）
- 时段 1：6月11日 20:00 ~ 08:00（夜间）
- 时段 2：6月12日 08:00 ~ 20:00（白天）
- ...
- 时段 19：6月21日 08:00 ~ 20:00

## 八、完整调用流程（伪代码）

```python
import httpx
from datetime import datetime, timedelta

# 1. 准备配置
API_URL = "http://10.226.120.112:8088/tjgrid/gdyb/getGdybDataByParam"

coord_map = {
    "天津市区": "117.14_39.24",
    "蓟州": "117.45_40.05",
    "宝坻": "117.28_39.73",
    # ... 其他区域
}

element_map = {
    "WEA": "天气现象",
    "TMAX": "最高气温",
    "TMIN": "最低气温",
    # ... 其他要素
}

# 2. 计算起报时间
now = datetime.now()
if now.hour >= 8:
    fcst_time = now.strftime("%Y%m%d080000")
else:
    yesterday = now - timedelta(days=1)
    fcst_time = yesterday.strftime("%Y%m%d200000")

# 3. 解析区域（根据用户输入）
region_text = "西青区"   # 实际来自用户查询
if "全市" in region_text or region_text == "":
    coords = list(coord_map.values())
else:
    coords = []
    for region_name in coord_map.keys():
        if region_name in region_text:
            coords.append(coord_map[region_name])
    if not coords:
        coords = list(coord_map.values())

lons = [c.split('_')[0] for c in coords]
lats = [c.split('_')[1] for c in coords]

# 4. 选择气象要素
elements = ['WEA', 'TMAX', 'TMIN', 'EDA', 'RHMAX', 'RHMIN', 'TP1H']

# 5. 组装请求参数
params = {
    "fcstTime": fcst_time,
    "element": ",".join(elements),
    "lon": ",".join(lons),
    "lat": ",".join(lats),
    "mode": "GDMODE",
    "startPeriod": "0",
    "endPeriod": "240",
    "interval": "12",
    "count": "0",
    "stationType": "3"
}

# 6. 发起请求
async with httpx.AsyncClient(timeout=120.0) as client:
    response = await client.get(API_URL, params=params)
    response.raise_for_status()
    result = response.json()

# 7. 解析返回数据
location_map = {v: k for k, v in coord_map.items()}  # 反向映射

result_data = result.get("resultData", {})
for coord, data in result_data.items():
    location = location_map.get(coord, coord)
    print(f"\n【{location}】:")

    for element_key, element_name in element_map.items():
        if element_key in data:
            values = data[element_key][0]
            valid_values = [v for v in values if v not in [9999.0, "--", None]]
            if valid_values:
                print(f"  {element_name}: {valid_values[:3]}...")
            else:
                print(f"  {element_name}: 暂无数据")
```

## 九、常见问题与调试

### 9.1 返回的数据全是 `9999.0` 或 `--`

**原因**：`fcstTime` 使用了尚未生成数据的时间点。

**解决**：尝试以下时间点依次测试：
- `当天 08:00` → `20260611080000`
- `前一天 20:00` → `20260610200000`
- `前一天 08:00` → `20260610080000`

### 9.2 请求超时或连接失败

**原因**：内网环境或网络问题。

**解决**：
- 检查服务器能否访问 `10.226.120.112:8088`
- 检查是否需要配置代理
- 增加超时时间（建议 120 秒以上）

### 9.3 区域匹配失败

**原因**：用户输入的区域名与映射表中不完全一致（如"河西区"不在映射表中）。

**解决**：默认回退到查询全市，或在映射表中增加更多别名。

### 9.4 HTTP 状态码 500

**原因**：参数格式错误或服务器端问题。

**解决**：检查参数是否缺失、经纬度是否配对、逗号是否为英文。
