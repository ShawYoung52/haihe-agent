# 前端用户体系对接说明

## 1. 目标

前端需要基于后端现有用户体系完成以下能力：

1. 登录
2. 注册普通用户
3. 管理员用户管理
4. 按角色展示不同功能入口

当前角色只有：

- `admin`：管理员
- `forecaster`：预报员
- `external`：外部用户

---

## 2. 页面结构建议

### 2.1 登录页

**作用**

- 用户输入用户名和密码
- 登录成功后进入主页面
- 根据角色显示不同功能入口

**交互流程**

1. 输入用户名、密码
2. 点击登录
3. 调用后端登录能力
4. 登录成功后读取用户角色
5. 按角色跳转或显示对应菜单

**登录后角色处理建议**

- `admin`：进入管理员视图
- `forecaster`：进入预报员视图
- `external`：进入普通用户视图

---

### 2.2 注册页

**作用**

- 普通用户注册
- 可选择角色，但不能选择管理员

**交互流程**

1. 输入用户名、密码
2. 选择角色
3. 提交注册
4. 注册成功后提示登录

**角色选择建议**

下拉只保留：

- 外部用户（`external`）
- 预报员（`forecaster`）

不允许展示管理员选项。

---

### 2.3 管理员用户管理页

**作用**

仅管理员可见，用于管理系统用户。

**建议模块**

- 用户列表
- 创建用户
- 修改状态
- 重置密码

**页面结构建议**

- 顶部：搜索框、刷新按钮、新建用户按钮
- 中间：用户表格
- 右侧或弹窗：新建用户 / 重置密码 / 修改状态表单

---

## 3. 接口调用流程

### 3.1 注册普通用户

调用：`POST /api/v1/auth/register`

**请求体示例**

```json
{
  "username": "user1",
  "password": "123456",
  "role": "external"
}
```

**前端行为**

- 成功后提示“注册成功，请登录”
- 跳转回登录页
- 不要允许注册 `admin`

---

### 3.2 获取用户列表

调用：`GET /api/v1/admin/users`

**前端行为**

- 仅管理员登录后展示该入口
- 页面进入时自动拉取列表
- 点击刷新重新请求

**表格建议显示字段**

- 用户名 `username`
- 角色中文名 `role_label`
- 状态 `status`
- 创建时间 `created_at`
- 更新时间 `updated_at`

---

### 3.3 创建用户

调用：`POST /api/v1/admin/users`

**请求体示例**

```json
{
  "username": "forecaster1",
  "password": "123456",
  "role": "forecaster"
}
```

**前端行为**

- 表单提交后刷新列表
- 默认角色建议为 `external`
- 可切换为 `forecaster`
- 不显示 `admin`

---

### 3.4 修改用户状态

调用：`PATCH /api/v1/admin/users/{username}/status`

**请求体示例**

```json
{
  "status": "disabled"
}
```

**前端行为**

- 表格里放一个“禁用 / 启用”按钮
- 如果当前状态是 `active`，按钮显示“禁用”
- 如果当前状态是 `disabled`，按钮显示“启用”
- 操作后刷新列表

**注意**

- 默认管理员 `admin` 不允许禁用

---

### 3.5 重置密码

调用：`POST /api/v1/admin/users/{username}/reset-password`

**请求体示例**

```json
{
  "password": "newpass123"
}
```

**前端行为**

- 在用户列表中提供“重置密码”按钮
- 点击后弹窗输入新密码
- 提交后刷新列表

---

## 4. 权限展示规则

### 4.1 `admin`

展示内容：

- 用户管理入口
- 所有普通功能
- 预报相关功能

### 4.2 `forecaster`

展示内容：

- 预报相关功能
- 普通功能

不展示：

- 用户管理入口

### 4.3 `external`

展示内容：

- 普通功能

不展示：

- 用户管理入口
- 预报员专属功能

---

## 6. 返回字段说明

### 6.1 注册 / 创建用户返回

```json
{
  "code": 200,
  "data": {
    "username": "user1",
    "status": "active",
    "role": "external",
    "role_label": "外部用户"
  },
  "message": "success"
}
```

字段说明：

| 字段 | 说明 |
|---|---|
| username | 用户名 |
| status | 用户状态 |
| role | 角色值 |
| role_label | 角色中文名 |

---

### 6.2 用户列表返回

```json
{
  "code": 200,
  "data": [
    {
      "username": "admin",
      "role": "admin",
      "role_label": "管理员",
      "status": "active",
      "created_at": "2026-06-22 10:00:00",
      "updated_at": "2026-06-22 10:00:00"
    }
  ],
  "message": "success"
}
```

---

## 7. 默认管理员

系统启动后默认存在：

- 用户名：`admin`
- 密码：`admin123`
- 角色：`admin`
- 状态：`active`

前端可用于首次登录验证。

---

## 8. 前端实现注意点

1. 不要把管理员账号放到注册页里
2. 用户管理页只给管理员看
3. 用户状态变化后要刷新列表
4. 重置密码后建议提示“需要重新登录”
5. 前端保存登录态时要带上角色信息，方便路由鉴权

---

## 9. 推荐的页面流转

### 登录成功后

- `admin` → 管理页 / 主界面
- `forecaster` → 预报员工作台
- `external` → 普通用户主页

### 管理员进入用户管理页后

- 先加载列表
- 可新增用户
- 可禁用/启用用户
- 可重置密码

---

## 10. 对接优先级建议

如果前端时间有限，建议按这个顺序做：

1. 登录页
2. 注册页
3. 用户列表页
4. 创建用户弹窗
5. 禁用/启用按钮
6. 重置密码弹窗

这样可以最快形成可上线闭环。
